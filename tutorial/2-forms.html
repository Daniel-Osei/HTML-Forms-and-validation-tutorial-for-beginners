HTML Forms

Forms are on of the most important part of every website, they allow to interact with users, and get information from here.

Let's start with basic HTML setup:


  <!DOCTYPE html>
  <html lang="en">
  <head>
      <meta charset="UTF-8">
      <meta name="viewport" content="width=device-width, initial-scale=1.0">
      <title>Forms</title>
  </head>
  <body>
      
  </body>
  </html>

Note: You can generate this setup using Emmet by typing ! (exclamation mark) and pressing Tab key.

Now let's create a basic form. Imagine that we want to create a registration form.

Inputs and labels


<form>
  <input>

Let's stop here for a second and note 2 things:

  • Input elements are self closing tags
  • If we want to label it we have to use <label> tags

Name and password inputs with labels:


  <form>
    <label>Name</label>
    <input>
    <label>Password:</label>
    <input>
  </form>

Since they are non-block element (displays next to each other) let's wrap them with div. 


<div>

    <input>
</div>
<div>

Submit

Finally we need some button to submit it:


  <button>Submit</button>

Now when we submit our form it will send data refresh page and send data to the same page.

In 99% cases you will want to submit your data either to other page or to handle your form using JS, so how to send data to other page?


  <form action="results.html">

This way we will submit our form to page results.html (it doesn't exist yet)

We also want to add another attribute which will specify which method we want to use to submit our data: 


  <form action="results.html" method="GET">

We covered this in our previous AJAX tutorial.

Let me quickly remind you that we could use either GET or POST method. In order to handle POST method we would need some server (i.e. Apache) and backend language (i.e. PHP). Therefore we will use GET method.

Let's create a results.html file now and paste the following content into it:


  <!DOCTYPE html>
  <html lang="en">
  <head>
      <meta charset="UTF-8">
      <meta name="viewport" content="width=device-width, initial-scale=1.0">
      <title>Results</title>
  </head>
  <body>
      <div id="results"></div>
      <a href="form.html">Back to results</a>
      <script>
          const results = document.getElementById('results');
          new URLSearchParams(window.location.search).forEach((value, name) => {
              results.append(`${name}:${value}`);
              results.append(document.createElement('br'));
          })
      </script>
  </body>
  </html>

Labels, except for helping us to describe given input have another important role. When we click on it they should automatically set focus on a field which they are referring to.

In order to achieve that we have to:

  1. Add new attribute for
  2. Add an id attribute to our input
  3. Make sure that for an id values matches. 


  <label for="name">Name</label>
  <input name="name" id="name">

Now when we click on the label it will automatically highlight corresponding input. This is also important for accessibility

Value and placeholder

Sometimes we want to add some default value to our form. Imagine that you have some settings page where use have specified his own email. When you load this page you want to pre-fill input with his existing email.

In order to do that you cane use another attribute - value

Some people may also want to use it this to suggest user what he should put in the input but this is not a good practice simply because when you use value, user have to delete your default value first before he/she can insert it's data.

In order to suggest user what input should be you can use another attribute - placeholder

As you can see , placeholder immediately disappears when you start typing. What is also important - a placeholder, unlike "value" isn't submitted to form data when you submit the form.
