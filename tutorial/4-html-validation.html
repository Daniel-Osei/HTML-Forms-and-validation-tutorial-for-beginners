HTML Validation

As we mentioned in a previous lesson, some inputs offers you a built in validations:

E-mail

Test the following code:


    <form action="results.html" method="GET" enctype="multipart/form-data">
        <div>
            <label for="email">E-mail</label>
            <input type="email" name="email" id="email">
        </div>
        <button type="submit">Submit</button>
    </form>

When you use email type, browser will check whether provided string has @ in it. Not to the best validation, that's why we will learn how to do a proper email validation using JavaScript soon:

Min / Max

As you remember, when we used min / max value browser will also take care of it:

Min/Max length

We can also specify minimum length of provided data:


    <div>
        <label for="name">Name</label>
        <input name="name" id="name" type="text" minlength="5" maxlength="25" placeholder="Min 5 chars">
    </div>

Cardinality

We can easily specify which fields are mandatory adding required attribute:

Pattern

We can also define allowed patterns: 


    <div>
        <label for="tel">Telephone</label>
        <!-- 4 digits pattern  -->
        <input name="tel" type="tel" pattern="[0-9]{4}" placeholder="####">
    </div>

This wil ensure 4 digits pattern.
