Input types file

HTML comes with a range od predefined input types to allow us accept any input from the user.

Text

All inputs which we've been using so far didn't have type specified. The default type is "text" 


    <input name="name" id="name" type="text" placeholder="First name" >

Password

When it come to password, you don't want to show typed chars on the screen.

In order to hide them we can use input type of password:

Number

When it comes to numeric data we can use input type of number:


    <div>
        <label for="age">Age:</label>
        <input name="age" type="number">
    </div>

This will show a special input with keys (up and down) and also allow us to use arrow keys to change the input.

While using number input we can also specify a minimum and maximum value:


    <input name="age" type="number" min="20" max="100">

Now when use try to submit value outside the allowed range it will show him an error.

We will learn more about validation in the next lesson - HTML validation.

We can also specify a step which will be used when user uses arrows or keyboard:


    <input name="age" type="number" min="20" max="100" step="5">

Date

Another input type is date:


    <div>
        <label for="date">Date:</label>
        <input name="date" type="date">
    </div>

This allows use to type in date, or choose it from a small date picker:

Similarly to number we can specify min and max date. This will grey out date from calendar and doesn't allow us to submit. If user type in date outside the allowed range it will show error on submit.

Checkboxes

When we want user to choose from predefined options we can use checkbox, i.e.


    <div>
        <label for="Pizza">Pizza</label>
        <input type="checkbox" name="pizza" id="pizza">
    </div>
    <div>
        <label for="Pasta">Pasta</label>
        <input type="checkbox" name="pasta" id="pasta">
    </div>

Note that user can check both options or none.

Radio

Sometimes we want users to choose only one option, in that case we can use radio type:


    <div>
        Gender
        <div>
            <label for="male">Male</label>
            <input type="radio" name="gender" id="male" value="male">
        </div>
        <div>
            <label for="female">Female</label>
            <input type="radio" name="gender" id="female" value="female">
        </div>
    </div>

NOTE! When we use radio, we have to provide the same name to both inputs so they know that these options are related to each other!

Since we can have only gender, we want user to choose only one option. When he/she change it's choice, the other radio automatically gets unchecked.

Another important thing to remember is that when you use radio inputs , you have to specify a value

Select

Another very common element is select. Note that this is not a input tag.

Example:

Note! You can specify a label of the option in two ways:

  • In between <option>myValue</option> tags
  • Using label attribute: label="Red"

Note! Similarly to radio buttons - you also have to specify a value attribute for options. Otherwise you will get empty result

While using select, you can also add multiple attribute which allows to select more than one option using a CTRL key button.


    <select name="eyeColor" id="eyeColor" multiple>

As you can see this will also change the way select looks like.

Textarea

Sometimes we want let user to provide a more text, in order to do that we can use <textarea></textarea> tags.


    <div>
        <label for="desc">Description</label>
        <textarea id="desc" name="desc"></textarea>
    </div>

As you seen, user can resize it , but we can also specify it size either using CSS, or attributes cols and rows:

Note that textarea accepts all white chars , so whatever you put in between tags will be rendered - even extra new lines.

FileInput

Another important input type is one which allows us to select files:


    <div>
        <label for="file">File</label>
        <input id="file" type="file" name="file">
    </div>

In order to make it working, we have to add enctype property to our form: 


    <form action="results.html" method="GET" enctype="multipart/form-data">

This will tell browser to send form in chunks since files might be big and require to be send in few parts.

We will have another tutorial on how to handle files on the backend file to really transfer it to our application.

Others

There are bunch of other input types like email, tel or url which are very helpful when you browse website on a mobile device since they display a dedicated keyboard. I.e. for tel type, there a phone will display a keyboard with only numbers and signs, without characters. 


    <div>
        <label for="phone">Phone</label>
        <input type="tel" name="phone" id="phone">
      </div>
      <div>
        <label for="url">URL</label>
        <input type="url" name="url" id="url">
      </div>
      <div>
        <label for="email">Email</label>
        <input type="email" name="email" id="email" required>
      </div>
